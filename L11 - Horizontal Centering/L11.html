Method 1: text-align: center

Best for: Inline and inline-block elements (text, images, buttons)
Centered with text-align
.parent { text-align: center; } .child { display: inline-block; }

‚úÖ Pros:

  • Simple and widely supported
  • Great for text and inline elements
  • Works in all browsers

‚ùå Cons:

  • Only works for inline/inline-block elements
  • Affects all child elements
  • Limited to horizontal centering only

Method 2: margin: 0 auto

Best for: Block elements with a defined width
Centered with margin auto
.child { width: 200px; /* Must have a defined width */ margin: 0 auto; }

‚úÖ Pros:

  • Classic and reliable method
  • Works well for content blocks
  • Excellent browser support

‚ùå Cons:

  • Requires a defined width
  • Only horizontal centering
  • Doesn't work with inline elements

Method 3: Flexbox (justify-content: center)

Best for: Modern layouts, both horizontal and vertical centering
Centered with Flexbox
.parent { display: flex; justify-content: center; align-items: center; /* Also centers vertically */ }

‚úÖ Pros:

  • Very flexible and powerful
  • Can center both horizontally and vertically
  • Works with any element type
  • Great for responsive design

‚ùå Cons:

  • Not supported in very old browsers
  • Can be overkill for simple centering

Method 4: CSS Grid (place-items: center)

Best for: Grid layouts and when you need precise control
Centered with Grid
.parent { display: grid; place-items: center; } /* Alternative syntax: */ .parent-alt { display: grid; justify-items: center; align-items: center; }

‚úÖ Pros:

  • Very concise syntax
  • Centers both horizontally and vertically
  • Great for complex layouts

‚ùå Cons:

  • Newer CSS feature
  • Might be overkill for simple centering

Method 5: Absolute Position + Transform

Best for: When you need to center over other content or create overlays
Centered with Transform
.parent { position: relative; } .child { position: absolute; left: 50%; top: 50%; transform: translate(-50%, -50%); }

‚úÖ Pros:

  • Works regardless of element size
  • Creates overlays effectively
  • Centers both horizontally and vertically

‚ùå Cons:

  • Removes element from document flow
  • Can cause rendering issues
  • Transform not supported in very old browsers

Method 6: Absolute Position + Negative Margins

Best for: Legacy browser support when element dimensions are known
Centered with Margins
.parent { position: relative; } .child { position: absolute; left: 50%; top: 50%; width: 200px; height: 60px; margin-left: -100px; /* Half of width */ margin-top: -30px; /* Half of height */ }

‚úÖ Pros:

  • Works in very old browsers
  • Precise positioning control

‚ùå Cons:

  • Requires knowing exact dimensions
  • Not responsive
  • Maintenance nightmare if sizes change
  • Removes element from document flow

Method 7: display: table-cell

Best for: Legacy browser support and when flexbox/grid aren't available
Centered with Table-cell
.parent { display: table-cell; text-align: center; vertical-align: middle; } .child { display: inline-block; }

‚úÖ Pros:

  • Works in older browsers
  • Centers both horizontally and vertically
  • Good fallback method

‚ùå Cons:

  • Semantically awkward (not actually a table)
  • Limited layout flexibility
  • Can interfere with other positioning
